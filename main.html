  • Number of Populations:
    Sets the number of populations to observe in the simulation routine.
    Takes an integer value between 1 and 10 (inclusive).
  • Population Size:
    Sets a population's fixed size in the simulation routine. Takes an integer value between 50 and 500 (inclusive).
  • Number of Allele Types:
    Sets and limits the number of possible allele types in a population. Takes an integer value between 2 and 8 (inclusive).
  • Mutation Denominator:

    Sets the denominator of an allele's mutation rate. An allele currently has
    chance to mutate into a different allele type. Takes an integer
    value between 1 and 1000 (inclusive).
  • Number of Migrants:
    Sets the number of members who migrate each generation. Takes an integer value between 0 and (inclusive).
  • Initial Number of Simulations:
    Sets the number of simulations to conduct before the user can interact with the Output page. Takes an integer value between 1 and 1000 (inclusive).
  • Simulation Rate:

    Sets the timer rate of executing consecutive simulations. The user interruption button will be more responsive if the Simulation Rate is set to a high value. Takes a value between 0.01 and 2.00 seconds (inclusive).

The mutation scheme used is based on the stepwise mutation model proposed by Kimura & Ohta (1978), with a minor modification to respect the bound- aries. For example, there are five possible alleles in a population, Alleles A, B, C, D, and E. Allele B can mutate to either Allele A or Allele C, and a similar process applies to Allele D, and Allele E. However, Allele A and Allele E can only mutate to Allele B and Allele D respectively.

Kimura, M., & Ohta, T. (1978). Stepwise mutation model and distribution of allelic frequencies in a finite population. Proceedings of the National Academy of Sciences, 75(6), 2868â€“2872.
The migration scheme uses simple random sampling to select a fixed number of migrants. These migrants then immigrate to a randomly chosen population, excluding their population of origin. As a result, populations before breeding are not necessarily the specified population size. However, this simple scheme is computationally quick to execute as there are no constraints to inform. Since the end users are typically going to be students, implementing a more complex migration scheme that might significantly increase the simulation time, which may frustrate the end user.
